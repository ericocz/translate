01

操作步骤

分两条通道搭建，互不依赖。通道一全链路同名——Stripe 与 Payoneer 必须同一人真实身份，否则资金卡在中转无法入账；通道二 YunGouOS 用本人身份证签约官方商户。

0

备齐材料

  • 通道一（Stripe）：大陆护照（KYC）、香港手机号（淘宝买 HK SIM，约 30–40 元）、本人国内银行卡（收结汇款）
  • 通道二（YunGouOS）：大陆身份证、本人手机号、本人国内银行卡（收结算款）、一个正规网站地址（进件需要）
  • 两通道的银行卡都建议用本人同一张或同名卡，对账省心
1

通道一 ① 先开 Payoneer，拿到收款账号

  • 打开 www.payoneer.com，用本人真实身份注册（注册免费）
  • 申请收款账户（美元 / 港币），Payoneer 会签发对应的收款账号，作为 Stripe 的提现目标账户
  • 记下这个账号，第 2 步绑定 Stripe payout 时要用
同名规则：Payoneer 账户只接受同名入账。你用哪个身份开的 Stripe，就只能收来自同名 Stripe 的款。先把这个账户开好，姓名以它为基准对齐 Stripe。
2

通道一 ② 注册 Stripe 香港账户

  • 打开 dashboard.stripe.com/register 注册
  • 国家选 Hong Kong，商家类型选 individual / sole proprietorship
  • 用护照过 KYC，填香港手机号
  • Payout 银行账户绑定第 1 步的 Payoneer 收款账号
  • 把提现设为手动（manual payout），不要自动
为什么手动提现：自动提现会把余额全部提走，一旦发生退款/争议而账户余额不足，Stripe 要你补款，容易触发封号。手动提现、定期小批提，更安全。
3

通道二 ③ 注册 YunGouOS，个人签约支付宝/微信

  • 打开 www.yungouos.com，注册并提交个人进件资料（身份证、本人银行卡、网站地址）
  • 分别完成微信支付与支付宝的官方签约（约 30 分钟人工审核，通过后即可发起交易）
  • 资金由微信 / 支付宝官方直连结算到你本人银行卡（微信 T+1），平台不碰钱
  • 务必先确认当前费率（开通费 + 平台服务费 + 官方 0.6%），口径以官网 / 客服为准
注意：个人进件有交易限额和风控；支付宝若被认定违规可能调整结算周期。别把唯一收款通道押在一家服务商上。
4

配置商品（两通道：买断 + 灵活充值）

  • 通道一 Stripe：建买断商品 $9.99（one-time，海外 / 信用卡；海外只做买断、不做充值，固定费吃掉小额）
  • 通道二 YunGouOS：建买断 ¥49.9 + 充值档 ¥1 / 5 / 10 / 20 / 100（无固定费，小额友好）
  • 买断与充值都须先注册邮箱账号，付费余额绑账号防浏览器漂移丢钱；消费时赠送额度（¥2）优先扣减
  • 前端把两种付款方式并列展示给所有用户、自由选择——不做地区强制分流
5

技术集成（双入口 + 统一发额度）

  • 前端：Next.js 提供两个支付入口（Stripe Checkout / YunGouOS 下单）
  • 后端：FastAPI 接两套回调——Stripe 的 checkout.session.completed 与 YunGouOS 的异步通知
  • 两套回调都验签，最终调用同一个"解锁 / 发额度"函数，落到同一套账本
  • 务必做幂等处理（用订单号当幂等键，防止重复发放）
核心原则：两条通道只是"入口"不同，发放额度的核心函数共用一个（输入档位 → 发 credits + 幂等键）。验签 + 幂等是发卡逻辑最容易出资损的地方。
6

两条通道分别实测

  • 通道一：信用卡真实下单 → Stripe 余额 → 手动提现到 Payoneer → 结汇到国内卡
  • 通道二：支付宝 / 微信真实付一笔小额 → 确认官方直连到本人银行卡
  • 两条都确认到账时效、费用、额度发放无误，再正式上线

付款方式（两种并行）

两种方式同时摆给所有用户、自由选择，不做地区强制分流——IP 判断不可靠（用户常挂代理），且大陆留学生身在海外也有支付宝 / 微信。给用户选择本身也能提升转化。

通道一信用卡 / 海外
用户→信用卡→Stripe HK→Payoneer→国内卡
通道二支付宝 / 微信
用户→支付宝/微信→YunGouOS（官方直连）→本人银行卡

两条通道汇入同一套账本

两种付款方式只是"入口"不同，最终都映射到内部统一的额度 / 解锁状态（credits）。两边回调收到付款后调用同一个发放函数，账本只认 credits 一个单位，前端按付款方式展示对应价格。

实现要点：Stripe 和 YunGouOS 各有一套回调，但发放逻辑共用一个函数（输入档位 → 发 credits + 幂等键）。新增 / 调整档位时两条通道一起改，不会算错。
02

成本明细

两条通道成本结构完全不同。费率会变动，以各平台后台实时为准；下面是当前参考值。

通道一 · Stripe + Payoneer≈ 7% + $0.30/笔

Stripe 国际卡约 3.9% + HK$2.35（≈$0.30），涉及货币转换再 +2%；Payoneer 结汇提现约 1.2%。有每笔固定费，小额极不划算，所以只用来收较大额的买断。退款 / 争议每笔有固定费且不退还。

通道二 · YunGouOS≈ 2.6%，无固定费

官方费率 0.6% + 平台服务费约 1–2%，综合约 2.6%，外加一次性开通费。没有每笔固定费，小额非常友好。资金官方直连结算到本人银行卡。具体开通费 / 服务费以官网为准（不同来源口径不一）。

小额场景 · 为什么充值走 YunGouOS

¥1 充值（YunGouOS ~2.6%）− ¥0.03
¥10 充值（YunGouOS ~2.6%）− ¥0.26
¥100 充值（YunGouOS ~2.6%）− ¥2.60
对比：¥10 走卡通道（固定费）− ¥3.2（32%）
结论小额必走 YunGouOS

分工很清楚：买断（较大额）两条通道都行，海外卡用户走 Stripe；小额充值只走 YunGouOS，卡通道的固定费会把小额吃掉三成。

03

风险与红线

两条通道各有风险点。下面是降低封号 / 冻结 / 资金风险的硬约束。

⚠ 照做能显著降低风险
  1. 通道一 · 全链路同名。Stripe 与 Payoneer 用同一人真实身份；Payoneer 只接受同名入账。
  2. 通道一 · 手动提现，别囤大额余额。定期小批提走，避免争议补款时余额不足触发风控。
  3. 通道一 · 控制退款率，ToS 灰色要有预期。用大陆身份开 Stripe HK 属规避地区限制，Stripe 有权按条款关停账户并冻结余额。
  4. 通道二 · 别把唯一通道押一家服务商。YunGouOS 等服务商可能倒闭跑路；好在资金官方直连（钱已到你卡），但接口会停、需换家重新进件。
  5. 通道二 · 个人进件有限额 / 风控。交易限额、支付宝违规可能调整结算周期；资料如实填写、控制异常交易。
  6. 两通道 · 税务自理。无论卡通道结汇还是国内收款，个人所得税申报都是你的责任，金额起来后提前规划。
