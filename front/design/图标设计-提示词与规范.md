# 工具栏图标重做 —— GPT Image 提示词与规范

> 本文件是「沉浸式翻译」工具栏图标重做的记录。
> 用 GPT Image(gpt-image-1 / GPT-4o 图像)生成,**不是 DALL-E**。
> 几何或配色改了,更新本文件后重跑生成 → 缩放 → 落 `public/icon/`。

---

## 〇、最终落地版(2026-06-07,以此为准)

经多轮迭代,**最终采用桃红双气泡翻译标(透明底,专为工具栏)**:

- **底**:**透明**(不铺满色块)——工具栏里干净、不像被硬塞的 App 图标。
- **图形**:两个**桃红圆角对话气泡**对角交叠,左上气泡白「**A**」、右下气泡白「**文**」(`A→文` 英译中)。气泡桃红 ≈ `#E9326F`(比 popup `--peach #D94A73` 略亮,同色系)。
- **主图处理**:裁掉透明边 → **强制拉伸到 1024×1024 填满**(裁后约 816×737,纵向略拉 ~11% 去掉上下留白),让 16px 尽量大。
- **四态各一套图**(角标用 ImageMagick 程序化绘制并烤进 PNG,右下角圆角方块 + 白字形):
  - **off** = 主图(无角标)
  - **打开翻译 / `translating`** = 琥珀 `#F5A623` 底 + 白「…」
  - **翻译成功 / `on`** = 绿 `#1FB24A` 底 + 白勾 ✓
  - **翻译失败 / `error`** = 红 `#E23B2E` 底 + 白叉 ✕
- 源图存档 `design/icon-src/{master,on,translating,error}.png`(`master-raw.png` 为作者原始件)。状态切图逻辑见 §五。

**重新生成配方**(几何/配色改了重跑;**角标 box `640,640 1018,1018`**,基于 1024 主图):
```bash
RAW=design/icon-src/master-raw.png
# 主图:裁透明边 → 强制拉伸填满 1024×1024(上下左右都顶满,无留白)
magick "$RAW" -fuzz 5% -trim +repage -resize 1024x1024\! -background none design/icon-src/master.png
M=design/icon-src/master.png
# on(绿勾)
magick "$M" -fill "#1FB24A" -draw "roundrectangle 640,640 1018,1018 66,66" \
  -fill none -stroke white -strokewidth 58 \
  -draw "stroke-linecap round stroke-linejoin round polyline 738,844 806,912 927,761" design/icon-src/on.png
# error(红叉)
magick "$M" -fill "#E23B2E" -draw "roundrectangle 640,640 1018,1018 66,66" \
  -fill none -stroke white -strokewidth 58 \
  -draw "stroke-linecap round line 753,753 905,905" -draw "stroke-linecap round line 753,905 905,753" design/icon-src/error.png
# translating(琥珀…)
magick "$M" -fill "#F5A623" -draw "roundrectangle 640,640 1018,1018 66,66" \
  -fill white -stroke none \
  -draw "circle 753,829 753,857" -draw "circle 829,829 829,857" -draw "circle 905,829 905,857" design/icon-src/translating.png
# 四套各缩 16/32/48/128(16/32 加 -unsharp);见 §四
```

> 角标只盖右下「文」一角,**「文」大部分露出**;off 主图完整 A+文。这是「角标够大能看清状态」与「保留文字」的平衡。

> ⚠️ 注意 `-strokelinecap` 选项本机 IM 不认,linecap/linejoin 必须写进 `-draw` 的 MVG 串里。

**下文 §一~§四 的「沉浸水线 / 桃红」及之前的「深绿双卡」均为历史方案,未采用,仅留作记录。**

## 一、设计定调(已与作者确认)

| 项 | 决定 |
|---|---|
| 用途 | **仅工具栏**(不做商店列表/popup 头部的 128px 大图) |
| 母题 | **沉浸水线** —— 扣产品名「**沉浸**式」,别人抢不走 |
| 风格 | **扁平**、大胆、有分量;纯色填充,无渐变/阴影/3D |
| 背景 | **实心、不透明**(整块桃红铺满画布,不是透明底浮一个字) |
| 配色 | **桃红双色阶**:上层亮桃红、下层深酱红、记号纯白 |
| 状态 | **三套位图**按态切图:off=主图 / on=on图 / **translating 复用 on 图** / error=error图(见 §五) |

**核心隐喻**:画面被一道白色「水线」横切。水线**之上**是英文原文(白色空心 `A`,浮在水面);水线**之下**是已「浸」成的中文译文(**实心白色汉字「文」**)。`A → 文` = 英译中,内容越过水线即被翻译。

```
╭──────────────╮
│      A        │  上层 亮桃红 #E0517A，白色空心 A（英文原文）
│┈┈┈┈┈┈┈┈┈┈┈┈│  白色水线
│      文       │  下层 深酱红 #A82B50，实心白「文」（中文译文）
╰──────────────╯
```

## 三、GPT Image 提示词(主用,英文)

英文提示词对 GPT Image 最稳。直接整段粘贴;**不要**开启透明背景(我们要实心底)。

```
Flat vector app icon, square, filling the ENTIRE square frame edge to edge,
with no outer margin and NO transparent background.

Concept: an "immersion waterline". The square is split horizontally across the
exact middle into two solid flat color fields: the TOP half is warm peach-pink
(#E0517A); the BOTTOM half is a deeper rose (#A82B50). A crisp thin WHITE
horizontal line runs straight across the middle where the two colors meet — this
is the "waterline".

ABOVE the waterline, in the peach-pink field, sits a single capital letter "A"
drawn as a thin WHITE OUTLINE (hollow), representing the original, untranslated
text floating on the surface.

BELOW the waterline, in the deeper rose field, sits the Chinese character "文"
(U+6587, meaning "text / written language") as a bold SOLID FILLED WHITE glyph,
representing the finished Chinese translation that has been immersed and
converted. Render "文" with correct, clean strokes.

Together they read as a translation mark: "A" (English) above the surface turns
into "文" (Chinese) below it.

Style: completely flat, bold, geometric, minimal. Solid flat colors only — no
gradient, no drop shadow, no 3D, no bevel, no texture, no realistic water, no
reflection. Thick strokes with rounded ends. Keep generous spacing so the mark
stays clear and legible when scaled down to 16x16 pixels.

Only these elements: the two color fields, the white waterline, the hollow "A"
above, the solid white "文" below. No other letters, no extra words, no numbers,
no watermark, no logo text. A SINGLE icon only — not a grid, not multiple variants.
```

### 关于「文」字渲染(重点)

GPT Image 渲染 CJK 偶有变形(多/少笔画、结构歪)。出图后**务必逐版核对「文」字形是否标准**:`文` = 一点、一横、一撇、一捺,共 4 笔。若变形,用编辑功能修,或**直接用字体合成正确的「文」**贴回(最稳)。这是整张图语义的关键,不能糊弄。

### 备选汉字

若想换字(语义略变):`中`(最粗壮、16px 最清晰,读「中文」)、`译`(最点题「翻译」但笔画多易挤)。把提示词里的 `"文"` 整体替换即可,本文件其余不动。

### 中文提示词(等价,备用)

```
扁平矢量 App 图标,正方形,图形铺满整个方形画面、四周不留白边、背景不透明。

母题「沉浸水线」:方形被正中一道横线一分为二,上半是实心亮桃红 #E0517A,下半是
实心深酱红 #A82B50,两色交界处有一道清晰的白色细横线,即「水线」。

水线之上(桃红区)正中,一个白色空心(描边)大写字母 A,代表浮在水面的英文原文。
水线之下(深酱红区)正中,一个粗壮的实心白色汉字「文」(笔画标准、结构端正),
代表已「浸」成的中文译文。二者合读即 A→文(英译中)。

风格:完全扁平、大胆、几何、极简;纯色填充——无渐变、无阴影、无 3D、无质感、
无写实水面、无倒影;笔画粗、端点圆;留足间距,缩到 16×16 仍清晰。

画面只有:两块色域、白色水线、上方空心 A、下方实心「文」。不要其他字母/多余文字/
数字/水印/logo 文本。只画一个图标,不要拼图、不要多个变体。
```

## 四、生成与落地流程

1. **生成**:GPT Image,`size 1024x1024`、`quality high`、`format png`、**背景不透明**(不要传 `background:transparent`)。一次出 3~4 张挑最利落的。
   - API 示例(可选):
     ```jsonc
     {
       "model": "gpt-image-1",
       "prompt": "<上面英文提示词>",
       "size": "1024x1024",
       "quality": "high",
       "output_format": "png"
       // 注意:不要加 "background":"transparent"
     }
     ```
   - ChatGPT 里直接粘提示词即可;若出现白边,追一句 "fill the whole square, no margin"。
2. **微调(大概率需要)**:这是精确排版型记号,GPT Image 不一定一次把空心 A、「文」字、水线摆正(尤其**「文」易变形**,见 §三)。用它的**编辑/局部重绘**修,或挑最接近的再编辑。始终不够锐/字不对时的兜底:用矢量/字体把 A + 水线 +「文」**合成**一版(更干净、字形最准),桃红底再铺上。
3. **缩放**:把 1024² 用 **Lanczos** 缩到 `128 / 48 / 32 / 16`。
4. **16px 体检**:细描边 A 与「文」在 16px 易糊。**16/32 版改用实心小 A**(弃描边)、并把「文」加粗/简化、加粗水线,保证笔画 ≥2px。「文」若在 16px 实在认不出,可只保留「A↘水线」+ 一抹实心,但 48/128 务必清楚出「文」。可单独出一版 16/32 专用图。
5. **落盘**:三套图各缩 4 尺寸 → `public/icon/`:主图 `{16,32,48,128}.png`、`on-*`、`error-*`;1024²/原图存档在 `design/icon-src/{master,on,error}.png`。
   - 缩放命令:`magick src -filter Lanczos -resize 128x128 -strip out`;16/32 追加 `-unsharp 0x0.7+1.0+0` 救细节。
   - 角标已烤进图,故 `lib/icon.ts` 只按态切图(见 §五),不画运行时角标。

## 五、状态实现(已落到 `lib/icon.ts`)

**最终方案:角标已烤进位图,按状态 `setIcon` 切图,不用任何运行时原生角标。**

| 状态 | 位图 | 含义 |
|---|---|---|
| **off** | 主图 `icon/{16,32,48,128}.png` | 此站不在自动翻译列表(双气泡主图,无角标) |
| **on** | `icon/on-*`(右下绿方块白勾 `✓` `#1FB24A`) | 翻译成功 / 已开启已译完 |
| **translating** | `icon/translating-*`(右下琥珀方块白「…」`#F5A623`) | 打开翻译 / 正在翻译 |
| **error** | `icon/error-*`(右下红方块白叉 `✕` `#E23B2E`) | 翻译失败 |

设计要点:
- 四态各一套位图,角标(圆角方块 + 白字形)用 ImageMagick 程序化绘制并烤进 PNG;`setTabIcon` 只按态 `setIcon` 切图、不画运行时角标(并清掉历史残留角标)。
- 角标右下角(box `640,640 1018,1018`,约占宽 37%),16px 能看清状态;只盖「文」一角、大部分露出(off 主图完整)。
- 图标透明底、拉伸铺满无留白、不灰化——off 也是满色双气泡主图,只是没角标。
- 四套源图 `design/icon-src/{master,on,translating,error}.png`,用 ImageMagick **Lanczos** 缩放,16/32 追加 `-unsharp` 锐化。几何变了就按 §〇 配方从 src 重生成。
