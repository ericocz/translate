沉浸式翻译 · 工具栏图标方案 V2

2026-06-14 · 三个方向，供你反馈迭代

为什么重做？ 现图标是「双气泡 + A/文」——这恰是 Google 翻译 / 微软翻译几乎所有翻译工具的通用图案，一看是「翻译」，却不知是「我们这款」，辨识度为零，且 16px 糊成色块。
这次的设计原点（反复权衡后锁定）：
  • 从主题色出发：翻译类工具清一色蓝/绿（Google、DeepL、微软），桃红是我们最强的差异化记忆点，当资产用足。
  • 魂在「沉浸」：整页就地翻译、原文垫着、译文逐块「浸」成中文——这是产品名、也是别人抢不走的隐喻。图案从这里长，而非再画气泡。
  • 16px 铁律：粗、实、高对比的单一剪影；细描边一律否（之前双线版正因「没存在感」被弃）。

桃红的策略价值：在一排蓝绿色的翻译工具里，桃红是唯一的暖色——用户扫一眼工具栏就能凭颜色定位，不必读图形。三套方案都把桃红作为唯一主色，原文灰仅作「未译」配角，状态色（琥珀/绿/红）只出现在右下角标，不与主色争夺记忆。

二 · 三个方向

方案 A · 最稳·最点题

沉浸水线

IMMERSION

「A」浮在桃红水面，「文」浸入水下。一道白色水线把英文原文与中文译文分隔——内容越过水线即被「浸」译。直接把产品名「沉浸」画成图，A→文 又明确传达「翻译」。

为何成立名字即图形，别人抢不走；A+文 一眼知是翻译。
16px上下两色块 + 反白「文」，对比足、剪影稳。

方案 B · 最贴产品行为

逐行浸译

FADE-IN

一整段文字，自上而下被桃红「浸」满。上两行已译（实心桃红）、中间一行正在译（桃红前沿推进、白色高光）、底行还是原文灰。画的就是产品独有的整页逐块淡入替换这一招。

为何成立把「就地、逐块、淡入」的核心交互直接做成图。
16px叠条剪影 + 上桃红下灰，像「正在翻译的段落」。

方案 C · 最具记忆点

译墨一滴

INK DROP

一滴桃红「译墨」，浸出反白「文」。墨滴 = 把整页「浸」透的那一滴译墨（呼应「沉浸/浸」）。剪影是最独特、最好记的单形状，颜色一拉满就是品牌，内嵌「文」点明翻译。

为何成立水滴剪影极易记、辨识度最高；颜色即品牌。
16px桃红实心水滴 + 反白「文」，远看也是一团醒目桃红。
