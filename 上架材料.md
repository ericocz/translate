# 沉浸式翻译 · 上架材料（Chrome / Edge）

> 提交应用市场所需的文案、权限说明、素材清单与回归口径。配套见 [`隐私政策.md`](隐私政策.md)。

## 一、商店文案

- **名称**：沉浸式翻译（DeepSeek V4 Flash）
- **类目**：生产力工具（Productivity）
- **语言**：简体中文（上架全球建议补英文）
- **简介（一句话）**：把英文网页整页翻成中文——原文先垫着、译文流式逐块替换，只显示译文。
- **详细描述（草案）**：
  - 面向技术阅读：整页翻译（正文/导航/按钮/页脚等可见文字全译），按 DOM 顺序自上而下。
  - 体验：页面加载即显示英文原文，译文到达后逐块淡入替换；Ctrl/⌘+点击某块可在中/英间切换。
  - 速度与成本：译文全局共享缓存，常见页面秒级命中；模型为 DeepSeek V4 Flash（关闭思考模式，首 token 更快）。
  - 账号：登录后免费畅用；未登录每天可免费翻译 3 个页面。
  - 隐私：只为翻译发送页面可见文本；运营打点仅记录域名与计数，不记录完整 URL/正文。

## 二、权限用途说明（提交时逐项填写）

| 权限 | 用途 |
|---|---|
| `storage` | 保存「自动翻译的网站」白名单、设置、匿名设备 ID、登录令牌。 |
| `commands` | 提供「翻译/取消翻译此网站」的键盘快捷键（默认 Alt+Shift+A / ⌘⇧A）。 |
| `webNavigation` | 监听单页应用（SPA）的同文档路由跳转（History API），对新路由自动重译。 |
| `host_permissions: <all_urls>` | 内容脚本需在用户开启翻译的任意网页上运行并替换文本；后台 service worker 需向本项目后端发起翻译/打点请求。**不在未开启的站点收集数据**。 |

> 说明：因含账号体系与运营打点，提交时需在「数据使用」声明中如实勾选所收集的数据类别（账号邮箱、用量统计），并提供隐私政策 URL。
