# 浏览器沉浸式翻译插件 · 用户体验设计

> 自用工具｜面向技术文章与英文网站阅读｜翻译模型 DeepSeek V4 Flash
> 本文档是体验层的设计基准，所有决策均已敲定，可作为界面设计与开发的依据。

---

## 一、设计哲学

这个插件的目标，是把整个英文网页变成一个可用的中文页面，而不仅仅是翻译文章正文。导航、侧边栏、按钮、页脚、评论，凡是页面上看得见的文字，都会被翻成中文，让你不只是读懂文章，也能在这个站点里自如地走动。

衡量它好不好的唯一标准，是它有多“隐形”。这里的隐形不是指功能少，而是指翻译要干净、不打扰、像这一页本来就是用中文写的。任何打断、闪烁、多余的界面元素，都是在向这个目标收税；每加一样东西，都要先问它配不配得上这笔税。整份设计里因此几乎看不到横幅、水印、进度条这类“吵”的东西。

---

## 二、翻译范围与激活

插件的默认姿态是安静的：没有被你标记过的网站，它不动手，页面保持英文原样。

激活模型极简，只围绕“域名”这一个粒度展开。工具栏图标是你和插件唯一的常驻接触点，它用自身状态告诉你当前站点是否已标记、此刻是否正在翻译、是否已译完，一眼即知，无需点开。在一个你常读的站点上点开图标、打开“在此站点自动翻译”，这个域名下的所有页面从此一打开就自动全页翻译。再点一次关掉，就是按域名取消翻译，且关闭的那一刻当前页面立即变回英文原文——因为原文始终保存在背后，切换是瞬间完成的。

这里没有“临时翻译一次”的入口。遇到一个不在白名单、又只看这一次的页面，需要先把它的域名加进来读、读完再关掉。这是为换取激活模型的极简而接受的取舍。

翻译范围是整页的全部可见文字，不区分内容类型，正文与界面一视同仁。藏在属性里的内容（图片 alt、输入框 placeholder 等）在第一版不翻，留待以后按需补充。

## 三、阅读流程：原文先垫，自上而下替换

打开一个白名单页面的瞬间，你看到的是完整、即时、没有任何布局跳动的英文原文。没有白屏，没有骨架占位，没有转圈。你可以立刻从头开始读，哪怕一个字的译文都还没回来。

随后译文流式回来，按页面的文档顺序自上而下，逐块把英文替换成中文。由于整页是一次性喂给模型、让它按顺序输出的，译文天然从顶部开始依次到达——通常顶部菜单会先于正文变成中文。替换动作要做得安静：用一个很短的淡入过渡让英文柔和地化成中文，而不是硬切，这样你专注阅读某一段时，余光里其它段落的变化不会扰人。

“原文先垫着”这个底色带来一个贯穿全局的好处：任何还没轮到、或翻译失败的内容，都只是静静地保持英文，不需要任何特殊的视觉处理。你顺着读下去，前方内容持续被译好，几乎永远不必盯着空白等待。

关于页面位移：中文与英文高度不同，上方段落被替换时下方内容会重排，理论上可能让你正读的行发生跳动。第一版不专门处理，依赖浏览器原生的滚动锚定机制在多数情况下自动钉住阅读位置；若实际使用中确有困扰，再做针对性优化。

---

## 四、译文呈现与样式保留

译完的页面应当让你忘记插件的存在：一篇干净的中文页面，加粗、斜体、链接、大字号等行内样式都准确地落在对应的中文词上，读起来像原作者用中文写的。

样式保留通过“占位标记”机制实现——抽取原文时，每一处内联样式被换成只带编号的成对标记交给模型，模型在翻译时按中文语序移动这些标记，客户端再依据自己保存的“编号到原始样式”映射把样式精确还原。具体样式（哪种加粗、链接的地址、字号大小颜色）始终留在客户端，不交给模型复述，因此不会丢失或被改坏。

有两类内容刻意保持英文：代码块和行内代码（如 `fetch()`）。对技术阅读而言，把代码翻成中文只会帮倒忙，保留原样才是对的。

整个译完状态里，唯一提示“这是翻译页”的，只有工具栏图标的状态；页面上没有水印、没有角标。

## 五、查看原文的两条路径

两条路径服务于不同粒度的需求，互为补充。

**点状：Ctrl + 点击单段。** 当你读到某一句觉得别扭、怀疑译错，按住 Ctrl 点击该段，这一段就地切换为英文原文并保持；再次 Ctrl + 点击，切回中文。它是粘滞式的，方便你停在原文上对照着读一会儿。

**面状：快捷键整页翻面。** 想整段、整节读英文，或对译文质量做通盘核对时，一个快捷键把整页在“全中文”和“全英文”之间瞬间翻面，再按一次翻回。快捷键可配置，以免与网页自身的快捷键冲突。

两条路径都因为英文原文始终保存在背后，所以切换都是瞬间、无需重新请求模型。

## 六、出错与异常

得益于“原文先垫着”，绝大多数异常都被无声地消化。

单段翻译失败（未返回，或占位标记校验不通过）时，该段就保持英文，看起来与“尚未轮到翻译”无异，不出现任何报错或空块。整篇流程结束后仍残留的英文段落即是失败项，控制面板提供“重试未完成的部分”一键补译；由于前缀缓存的存在，重试又快又几乎不耗费用。流式中途断开同理：已译的留中文，未译的留英文，重试接着补。

整页失败的提示要说人话、分清来源。考虑到通过 Clash 代理访问外部服务，相当一部分失败其实是代理那一刻没连通，而非接口问题。所以错误信息要区分“网络/代理未连通”与“模型接口报错”，让你一眼判断该去检查代理，还是单纯再点一次重试。

## 七、控制面板与设置

工具栏图标点开是一个紧凑的控制面板，只放高频项：当前站点状态、“在此站点自动翻译”开关、“重试未完成”、以及通往完整设置的入口。翻译进行时可显示一行极轻的进度文字（如“已翻译 12 / 40 段”），让你心里有数——是一行小字，不是进度条。

完整设置页放偶尔才碰的配置：白名单管理（查看与删除已开启的域名）、DeepSeek API Key、整页翻面的快捷键。不设翻译模式开关（只有通俗模式），不设术语表，悬停相关设置也不存在（已改为 Ctrl + 点击）。

## 八、第一版明确接受的限制

以下几点是已知的、第一版选择不处理的边界，列出以备日后判断：

翻译活的界面不如翻译静态正文稳定。按钮、菜单、动态组件可能被页面自身的 JavaScript 重新渲染，导致已替换的中文被打回英文，个别依赖文字内容的交互也可能出现异常。对偏静态的文档站影响很小，对应用型页面会比较飘，第一版接受此现状。

只处理页面加载时已存在的内容。异步渲染或“加载更多”后追加的内容，第一版不自动捕获翻译，留待真正需要时再补监听机制。

页面重排造成的滚动跳动不专门处理，依赖浏览器原生滚动锚定。

---

## 九、决策速查

翻译范围是整页全部可见文字，不区分内容类型。激活按域名粒度，一个开关控制开与关，关闭即刻还原英文，无一次性翻译入口。阅读流程是原文先垫、自上而下逐块替换、淡入过渡。样式靠占位标记保留，代码保持英文。查看原文有 Ctrl + 点击单段粘滞切换、与快捷键整页翻面两条路。只有通俗模式，无术语表。出错时失败段静默保留英文，提供重试，错误提示区分代理与接口。模型为 DeepSeek V4 Flash，非推理模式流式输出，稳定的系统提示词置于前缀以命中上下文缓存。
