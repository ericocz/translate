# 自动化测试目标网站清单

本文件收录 **150 个英文网站**，作为沉浸式翻译扩展的自动化/手动测试目标，分两批：

- **#1–#50（核心边界用例）**：不以流量为标准，刻意覆盖扩展**真正容易翻车的维度**，每站标注它主要考验哪条能力 / 铁律。
- **#51–#150（高流量 / 大陆高频）**：补充访问量大的英文站点，**侧重大陆用户高频访问**的英文站，扩大真实场景覆盖；额外标注「大陆访问（参考）」一列。

> 配套：踩坑案例见 [`翻译问题记录.md`](翻译问题记录.md)；实现基准见 [`CLAUDE.md`](CLAUDE.md)。
> 整理日期：2026-06-05。

## 为什么这样选（测试维度 → 对应风险）

| 维度 | 考验什么 | 对应铁律 / 已知坑 |
|------|---------|------------------|
| **渲染框架多样性** | 抽取/重建对各类 DOM 结构（Docusaurus / Next.js / Sphinx / mdBook / VitePress / SPA）是否稳健；wrapper 保形 | 翻译问题记录 #1（Docusaurus logo wrapper 被静默丢弃） |
| **React/Next.js hydration** | 替换 DOM 与 React 协调冲突、`removeChild` 崩溃、#418 告警 | CLAUDE.md 已知坑（react.dev 等） |
| **动态重渲染** | 页面自身 JS 把已替换的中文打回英文 | CLAUDE.md 已知限制 |
| **代码密度** | 代码块/行内 code 不被翻译 | 铁律 6 |
| **行内样式密度** | `<gN>`/`<xN>` 标记成对、编号合法、重建正确 | 铁律 3 |
| **整页文字** | 导航/按钮/页脚等非正文也翻 | 铁律 7 |
| **长页 + 多块** | 分批（~40 块）、流式回填、有限并发 | 流水线 3–4 |
| **付费墙/登录/地域** | 内容不全或被重定向时的鲁棒性（作者经 Clash 代理访问） | DeepSeek 接入要点 |

> **渲染/框架列为"参考值"**：基于公开常识标注，可能随站点改版变化；测试前以实际页面为准。

## ✅ 可访问性核对（2026-06-05，firecrawl 云端抓取）

用 firecrawl 从云端实地抓取了全部 50 个 URL，验证「在线可访问 + 内容为英文」。**云端抓取代表站点本身的状态，不代表你本地经 Clash 的连通性。**

- **URL 全部有效**：50/50 域名与路径正确，无拼写错误。
- **48 个直接抓到英文内容**，语言判定全部为英文（个别页面含"中文"等语言切换链接，CJK 占比 <0.5%，不影响判定）。
- **2 个被 firecrawl 政策性拒绝**（返回 *"we do not support this site"*）：**#21 纽约时报、#29 Reddit**。这是 firecrawl 不抓这类站，**不是**扩展的问题——扩展跑在你真实浏览器里（有登录态、正常 UA），这两个站照常能翻译，**仍保留为测试目标**，只是没法用这套云端脚本自动核对。

> 提示：少数站点**首页**抓到的字节偏小（Medium ~1.7KB、npm ~2.1KB、Redux 首页 ~2.8KB），是首页内容本就少 / 有订阅墙所致；测试翻译质量时请进入**具体文章 / 文档深页**，更有代表性。

核对脚本与抓取产物在 `.firecrawl-check/`（`results.tsv` 可访问性、`lang.tsv` 语言判定、`pages/` 原始 markdown），可重复运行。

## 使用说明

- **域名级白名单**：扩展按域名触发，把上表 URL 的域名加入白名单即可在该域全站生效。
- **保证英文内容**：部分站点会按地域本地化（如 MDN、Wikipedia），上表已尽量用 `en-US` 路径或 `en.` 子域；测试前确认页面确为英文。
- **代理注意**：作者经 Clash 访问，新闻/学术类（NYTimes、Nature、Reuters 等）若失败，先区分"代理未连通"与"接口报错"再排查（见 CLAUDE.md DeepSeek 接入要点）。
- **新发现的问题**：按 [`翻译问题记录.md`](翻译问题记录.md) 模板回填，并把该站加入上面的"优先回归"表。
